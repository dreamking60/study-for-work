# 每周交付检查清单

每周固定产出 6 项：
1. 需求清单（用户故事 + 验收标准）
2. 技术设计文档（架构图 + 时序图 + 数据模型）
3. 可运行代码（含启动脚本）
4. 测试结果（单测 + 集成测试）
5. 性能数据（压测参数 + 指标）
6. 周复盘（问题、根因、下周计划）

## 周末验收标准
1. 代码可跑通：新环境 10 分钟内可启动。
2. 文档可阅读：结构完整，可复现关键结论。
3. 指标可追溯：所有数字有测试或监控依据。
4. 质量可解释：能讲清“为什么这样设计”。

## 失败处理规则
1. 周目标未完成，不进入下一周，先补齐最低完成线。
2. 测试失败优先修复，不允许带红进入下周。
3. 文档缺失时不允许写简历成果。
