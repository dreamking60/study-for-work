# 学习执行标准（Definition of Done）

## 1. 每天完成标准
1. 学习输入：至少 1 个官方文档页面 + 1 条可运行示例。
2. 代码输出：当天必须有可运行代码，不允许只看不写。
3. 验收证据：至少 1 条命令输出记录到日报中。
4. 复盘记录：写下 1 个问题 + 1 个改进动作。

## 2. 每周完成标准
1. 功能完成：周目标中的核心功能跑通。
2. 工程质量：`go test ./...` 可通过。
3. 文档完整：需求、设计、压测、复盘齐全。
4. 可复现：新环境 10 分钟内按 README 跑通。

## 3. 质量红线
1. 没有验收命令输出，不算完成。
2. 没有测试，不算完成。
3. 没有失败记录与修复过程，不算真实项目经验。
4. 指标没有来源（压测/监控截图），不允许写进简历。
